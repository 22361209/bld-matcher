# Main Classic 视觉基线

## 用途

本文件记录 `main` 合并新设计前的整体视觉，便于在不重新设计页面的前提下立即切回。它只描述源码视觉基线，不包含数据库、上传文件、输出文件或环境配置。

## 可切换基线

- Git 分支：`main-classic-20260720`
- Git 标签：`main-classic-20260720`
- 固定提交：`e714dcf`（已包含 API Key 删除与 AI Agent 接入说明）

查看或临时运行经典界面时，在一个干净的工作树中执行：

```bash
git switch main-classic-20260720
```

这只切换源码，绝不覆盖 `data/`、`uploads/`、`outputs/` 或 `.env`。如需让已部署环境回到该界面，应在部署工作树切换到该分支后按正常 Git 部署流程重建服务；不要用数据同步或文件复制回退视觉。

## 经典 Main 的设计语言

- 浅冷灰画布（`#f5f7fa`）与白色表面，内容位于最大 1440px 的居中工作区。
- 蓝色 `#225d91` 用作选中导航、主操作和焦点，文本为深灰蓝；整体是克制、清晰的传统业务后台，而非品牌化仪表台。
- 顶部为轻量文字导航：浅底、细分隔线、6px 圆角，导航项可换行；管理员身份与退出操作在右侧。
- 每页保留 `workspace-header`：标题、说明和右侧状态/摘要先定义页面身份；其后依次是指标条、检索/操作区、数据表或表单。
- 工作区面板、输入框、表格和弹窗采用白底、细灰边、6px 圆角与轻阴影；信息密度中等，留白优先服务于扫描和录入。
- 询价、生产料单等首页遵循同一工作台节奏，不使用深色品牌导航、全宽工业画布或大幅展示型标题。

## 与新设计的边界

新设计改变的是页面壳、导航、间距、标题策略和共享视觉样式；领域服务、路由、权限、API、数据库和运行数据不以视觉基线为准。任何未来视觉回退都应使用本分支或标签，而不是手工反向修改页面。

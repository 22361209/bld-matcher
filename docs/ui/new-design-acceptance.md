# New Design 全站改造验收标准

本文件定义 `codex/new-design` 工作树的完成边界。改造目标是将现有页面统一为“精密工业工作台”视觉系统，同时保持业务行为、权限、数据安全和页面协议不变；发货通知与货物识别的 Web 页面按用户要求退役。

## 1. 范围

### 必须改造

- 登录页。
- 询价首页、匹配列选择、匹配结果。询价首页以 main 工作树的简洁入口为结构基准：标题与目录指标、单一且有界的号码/文件搜索控件、按需展开的历史记录；不增加右侧状态栏、流程卡或重复信息。真实查询、上传、权限、结果和历史记录继续保留。
- 报价记录及报价导入。
- 合同管理。
- 产品目录、产品编辑、目录导入预览、目录导出、批量图纸、产品数据同步。
- 管件列表及管件编辑。
- 生产料单、材料明细编辑、物料图纸。
- 账号管理、内部 API Key、操作日志、系统更新、业务数据同步。
- 全局导航、消息、按钮、表单、弹窗、指标、数据表格、分页、加载/空/错误状态。

### 必须退役

- 导航中不再出现“发货通知”和“货物识别”。
- `/shipping-notices`、`/shipment-recognition` 及其仅供 Web 页面使用的子路由不再对外提供页面能力。
- 删除对应模板和页面专用 CSS/JavaScript，并更新路由/OpenAPI 快照、测试和当前文档。
- 保留历史运行数据、上传文件、输出文件、数据库迁移、领域服务和命令行工具；本次不做数据删除。

## 2. 视觉系统

- 全站使用深石墨导航、冷灰画布、白色工作面、BLD 蓝主色和少量信号橙。
- 导航保持横向，桌面端不给宽表格增加固定侧栏。
- 每页必须形成“页面身份 → 主要动作 → 筛选/流程 → 内容 → 状态/分页”的稳定层级。
- 主操作每个区域最多一个实心强调按钮；次要操作使用描边或文字按钮。
- BLD/OE/合同编号等业务编码使用等宽或窄体数字风格，中文正文保持高可读。
- 不添加无业务意义的图标、渐变、统计卡或装饰图片。

## 3. 行为与兼容

- 现有搜索、筛选、排序、分页、标签页和视图状态继续写入 URL。
- 产品表格的列换序、列宽调整、筛选、固定操作列、图片/图纸预览和导出范围保持不变。
- 表单字段、名称、权限、CSRF、确认、错误映射和服务端业务规则保持不变。
- 弹窗支持 Escape、焦点圈定、关闭后焦点返回和可访问名称。
- 提交中、成功、空、错误、无权限、只读和长文本状态不因视觉改造退化。

## 4. 响应式与可访问性

- 1280×720 和 1440×900 下页面根节点无横向滚动；宽表格只在数据框体内部滚动。
- 760px 移动宽度下导航、主操作、表单和对话框可用，内容不遮挡操作按钮。
- 所有交互控件具备清晰的 `:focus-visible`；文本与背景对比满足常规业务界面可读要求。
- 长英文编号、中文名称、多行车型和错误信息不得撑破布局。

## 5. 工程协议

- 所有完整页面继续继承 `base.html`，保留唯一 `page_id` 和合法 `page_type`。
- 模板不新增内联脚本、内联样式或事件属性。
- CSS 继续按基础、共享组件、页面三层归属；不新增 ID 选择器或 `!important`。
- 不扩大 `policy/legacy_allowlist.json`，不引入新前端框架或构建流水线。
- 添加本次 `changes/*.json` 片段，并更新页面协议、项目简报和相关 ADR/运维文档中已失效的 Web 能力说明。

## 6. 验证门禁

- 静态协议与资产归属检查通过。
- 页面服务端渲染、权限和退役路由测试通过。
- `uv run python scripts/verify.py` 全部通过。
- 在隔离运行数据和独立端口上完成真实浏览器验收：登录、询价、产品目录、报价、合同、材料、管件、图纸、管理页、同步页。
- 询价首页在 1280×720 桌面尺寸下按 main 工作树的结构核对：搜索工作区居中且最大宽度为 640px，不出现右侧状态栏或流程卡；不得因全宽工作区而把搜索控件拉伸至难以扫读的宽度。
- 浏览器控制台无错误，页面 CSS/JavaScript 请求无 404，桌面和移动基准页面无根级溢出。
- `git diff --check`、提交范围审计和最终工作树清洁检查通过后方可 commit。

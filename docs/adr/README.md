# Architecture Decision Records

ADR 用于记录难以撤销或会影响多个领域的决定。新增框架、数据库、任务队列、AI供应商、公共 API 破坏性变化、数据所有权和页面协议变化都需要 ADR。

文件名使用 `NNNN-short-title.md`。状态包括 `proposed`、`accepted`、`superseded`、`rejected`。已接受 ADR 不改写结论；新决定通过下一份 ADR 替代。

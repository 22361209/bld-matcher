## 系统用途

这是一个 Flask 网页系统，主要用于：

1. 客户询价处理
2. 产品目录维护
3. 产品目录导入 / 导出
4. 产品图片和含税单价维护
5. 操作日志查看
6. 账号和权限管理
7. 生产料单生成
8. 冲压材料明细维护

## 当前主要页面

### 询价处理

入口：`/`

功能：

- 上传客户询价 Excel
- 支持 `.xls` 和 `.xlsx`
- 支持无表头文件，上传后可选择匹配列
- 支持一个单元格中多个 OE 号，用逗号、斜杠、分号、换行等拆分
- 不按普通空格拆分，因为部分 OE 号本身含空格
- 匹配后导出 Excel，保持客户源文件格式
- 输出文件新增 `BLD NO.` 和 `匹配说明` 列
- 文件名格式为 `re + 当前日期 + 源文件名`

### 产品目录

入口：`/products`

功能：

- 按 BLD 号搜索
- 按 OE 号搜索
- 启用 / 停用状态筛选
- 新增产品
- 编辑产品
- 停用产品
- 维护 OE 号、品牌号码、车型、含税单价、产品图片路径
- 显示产品图片缩略图，点击后在当前页面浮层预览
- 导入目录
- 导出目录
  - 按 BLD 号升序格式
  - 按汽车品牌格式
- 批量维护单价

产品目录导出时表头包含：

- `BLD NO.`
- `OE Reference`
- `Other Reference`
- `Product Image`
- `Unit Price`

导出字体要求：

- 中文：微软雅黑
- 英文：Arial

### 生产料单

入口：`/materials`

功能：

- 上传生产计划 Excel，生成冲压生产料单
- 下载生产计划模板
- 查看材料明细列表
- 按型号 / 编码搜索
- 启用 / 停用状态筛选
- 新增材料明细
- 编辑材料明细
- 停用材料明细
- 导入新的冲压料单 Excel，重建材料明细库

生产料单页面目前使用类似 Linear 的低对比紧凑风格，仅此页面单独改版，其它页面未同步此风格。

生成料单时现在读取数据库中的启用材料明细，而不是直接读取 Excel 文件。网页上编辑、新增、停用材料明细后，会影响下一次生成结果。

材料字段说明：

- 型号：用于匹配生产计划
- 编码：如 `KA8234`
- 类别
- 车型
- 零件名称
- 规格尺寸
- 下料只数
- 规格1：厚度，可有小数
- 规格2：宽度，单位 mm，整数显示
- 规格3：长度，单位 mm，整数显示

生产料单输出包含：

- `总汇总`
- `料单明细`
- `按规格汇总`
- `生产需求`
- 如有未匹配型号，额外生成 `检查提醒`

## 绝对不要随便覆盖的文件

### 最重要

`data/products.sqlite3`

这是当前系统的核心数据库，里面包含：

- 产品目录
- 账号
- 操作日志
- 人工映射
- 材料明细库

在 NAS 更新代码时，通常不要上传本地的 `products.sqlite3` 覆盖 NAS 上的数据库，否则 NAS 上新增或修改过的数据会丢失。

### 也要注意

`static/product_images/`

如果 NAS 上未来有更多产品图片，不要用本地旧图片文件夹整体覆盖。

## NAS 部署方式

NAS 使用 Synology Container Manager。

项目目录建议：

`/docker/bld-matcher`

`docker-compose.yml` 当前端口配置：

```yaml
ports:
  - "5055:8000"
volumes:
  - ./data:/app/data
  - ./uploads:/app/uploads
  - ./outputs:/app/outputs
```

这表示：

- NAS 访问端口是 `5055`
- 容器内部端口是 `8000`
- `data`、`uploads`、`outputs` 是持久化目录

### 更新 NAS 代码时推荐步骤

1. 在本地生成更新 zip
2. zip 中包含代码、模板、样式、Docker 文件
3. zip 中不要包含 `data/products.sqlite3`
4. 上传 zip 到 NAS 的 `/docker/bld-matcher`
5. 解压到当前文件夹，并覆盖同名代码文件
6. 确认 `/docker/bld-matcher` 下直接能看到：
   - `app.py`
   - `Dockerfile`
   - `docker-compose.yml`
   - `app/`
   - `templates/`
   - `static/`
7. 不要解压成 `/docker/bld-matcher/某个子文件夹/app.py`
8. Container Manager 中停止项目
9. 如有旧镜像导致不更新，可删除旧的 `bld-matcher-bld-matcher:latest` 映像
10. 重新构建项目
11. 启动项目

如果重新构建后还是老版本，重点检查：

- 当前打开的网址是否是 NAS 地址，而不是本机 `127.0.0.1`
- zip 是否解压到了错误的子文件夹
- Container Manager 是否仍然使用旧映像
- 浏览器是否缓存，可强制刷新

## 给新接手 Codex 的提示

如果在新电脑或新对话中继续开发，请先阅读本文件，然后检查：

```bash
pwd
rg --files
sed -n '1,140p' app.py
sed -n '1,220p' app/database.py
```

不要第一时间重构。这个项目是用户持续迭代出来的业务工具，很多设计来自实际使用习惯，优先保留已有行为。

修改前要注意：

- 不要覆盖 `data/products.sqlite3`
- 不要删除 NAS 上已有数据
- 页面更新后要重启 Flask 服务
- NAS 更新时要重建 Docker 映像
- 生产料单页面目前风格单独调整，不代表其它页面也必须同步
